About

o365enum is a powerful security research tool that enumerates valid usernames from Office 365 using ActiveSync, Autodiscover, or office.com login page methods. This tool helps security professionals identify valid user accounts during penetration testing and security assessments.

Quick Start

Usage

python3 o365enum.py -h
usage: o365enum.py [-h] -u USERLIST [-p PASSWORD] [-n NUM] [-v]
                   [-m {activesync,autodiscover,office.com}]

Office365 User Enumeration Script

optional arguments:
  -h, --help            show this help message and exit
  -u USERLIST, --userlist USERLIST
                        username list one per line (default: None)
  -p PASSWORD, --password PASSWORD
                        password to try (default: Password1)
  -n NUM, --num NUM     # of reattempts to remove false negatives (default: 3)
  -v, --verbose         Enable verbose output at urllib level (default: False)
  -m {activesync,autodiscover,office.com}, --method {activesync,autodiscover,office.com}
                        method to use (default: activesync)

Example

./o365enum.py -u users.txt -p Password2 -n 1 -m activesync

Output:
username,valid
nonexistent@contoso.com,0
existing@contoso.com,1

Enumeration Methods

ActiveSync Enumeration

This method sends Basic HTTP authentication requests to the ActiveSync endpoint. Instead of checking status codes, it analyzes the presence of the X-MailboxGuid header to determine if a username is valid.

  • Existing Account: Returns 401 with X-MailboxGuid header present
  • Nonexistent Account: Returns 401 without X-MailboxGuid header

Autodiscover Enumeration

Utilizes Office 365's Autodiscover service which may return different responses for valid and invalid users.

Office.com Enumeration

Tests usernames against the Office.com login endpoint to identify valid accounts.

Requirements
